\moduledescription{mod.tls.core.desc}

Module \tds{mod.tls.core} of subsystem \tdslink{sub.tls} implements the TLS
protocol. A TLS server is started to use the administration interface of the
TOE.

\moduleprocesses{prc.tls.core}

\moduleprocess{prc.tls.core.connect}{Opening a TLS Connection}

This process accepts an incoming TLS connection from a client. During TLS
handshake, ephemeral DH key exchange is performed. The reqired key pair is
generated by module \tdslink{mod.cryptsystem.keymgmt}.

\implementedsfr{\sfrlink{ftp_itc.1/tls} & \sfrlink{fcs_cop.1/tls.aes}}

\moduleprocess{prc.tls.core.auth}{Authenticating the Peer}

This proccess authenticates the peer. The peer's certificate is verified by
calling \tdslink[fq]{int.tls.cert.checkcert}.

\implementedsfr{\sfrlink{fcs_cop.1/tls.auth}}

\moduleprocess{prc.tls.core.disconnect}{Closing a TLS Connection}

This process closes the TLS connection. Upon closing the connection, the TLS
session keys are destroyed by calling
\tdslink{int.cryptsystem.keymgmt.destroykey}.

\implementedsfr{\sfrlink{ftp_itc.1/tls}}

\moduleinterfaces{int.tls.core}

\moduleinterfaceprovided{int.tls.core.accept}

This interface accecpts a TLS connection. It implements the TSFI \lslantls{}.

\moduleinterfaceprovided{int.tls.core.disconnect}

This interface closes a TLS connection.

\moduleinterfacerequired{int.tls.cert.checkcert}

The interface \tdslink[fq]{int.tls.cert.checkcert} is required for verifying the
peer's certificate.

\moduleinterfacerequired{int.cryptsystem.keymgmt.createkey}

The interface \tdslink[fq]{int.cryptsystem.keymgmt.createkey} is required for
creating the ephemeral DH key pair.

\moduleinterfacerequired{int.cryptsystem.keymgmt.destroykey}

The interface \tdslink[fq]{int.cryptsystem.keymgmt.destroykey} is required for
destroying the TLS session keys.
